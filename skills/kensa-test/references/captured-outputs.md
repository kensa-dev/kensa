# Kensa CapturedOutputs Reference

## Why CapturedOutputs?

`CapturedOutputs` hold values **generated by the system under test** that cannot be known before
the test runs — database sequence numbers, UUIDs, reference numbers assigned during processing.
They allow you to test the production application with real ID generators: capture the generated
value during the action phase, then use it in subsequent assertions (e.g. verify a notification
was sent for that exact id, or a row exists in a database under that exact key). This
precludes running the application with test/fake generators.

## Defining CapturedOutputs

**CapturedOutputs must always be defined inside a `CapturedOutputContainer` object and registered
during initialisation. Never define them elsewhere. Registration is what allows Kensa to give each
test invocation its own scoped output map — safe for parallel test runs.**

```kotlin
object TradeFinanceCapturedOutputs : CapturedOutputContainer {
    /** The LC number assigned by DocumentIssuanceService — system-generated, captured via onMatch. */
    val IssuedLcNumber = capturedOutput<String>("Issued LC Number")

    /** The notification ID returned by NotificationService — captured to verify async delivery. */
    val ApplicantNotificationId = capturedOutput<String>("Applicant Notification ID")
}
```

### Key rules

- Keys must be globally unique across all registered containers.
- Use descriptive keys like `"Issued LC Number"` not `"lcNum"`.

## Registering in the Test Class

Register in the `init` block alongside fixtures:

```kotlin
init {
    registerFixtures(TradeFinanceFixtures)
    registerCapturedOutputs(TradeFinanceCapturedOutputs)
}
```

## Capturing Values

The typical capture point is the `onMatch` lambda of a `ThenSpec`, which fires after a successful
assertion and receives the matched value:

```kotlin
private fun capturingIssuedLcNumber() = object : ThenSpec<LcApplicationResult> {
    override val collector = StateCollector<LcApplicationResult> { holder.result }
    override val matcher = beInstanceOf<LcApplicationResult.Approved>()
    override val onMatch: CollectorContext.(LcApplicationResult) -> Unit = { result ->
        outputs[IssuedLcNumber] = (result as LcApplicationResult.Approved).lcNumber
    }
}
```

You can also write directly inside an `ActionContext` action:

```kotlin
outputs[IssuedLcNumber] = response.lcNumber
```

## Reading Captured Values

`KensaTest` exposes `outputs` directly in the test body. The instance is scoped to the current
invocation — parallel test runs each get their own isolated copy. `outputs` is also available in
all context types — `GivensContext`, `ActionContext`, `CollectorContext`:

```kotlin
// StateCollector
private fun theIssuedLcNumber() = StateCollector { outputs(IssuedLcNumber) }

// Test body
and(theIssuedLcNumber()) { shouldBe(outputs(IssuedLcNumber)) }
```

Use `outputs.getOrNull(key)` when the value may be absent.
